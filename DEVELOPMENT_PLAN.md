# Tiptap + Yjs 协同富文本项目开发计划

该项目旨在基于现有的 Turborepo 全栈架构，开发一个支持多用户实时协作的富文本编辑系统，并配套完善的后台权限管理功能。

## 1. 基础架构搭建与环境准备（已部分完成）

- [x] 基于 Turborepo 划分 `apps/api` (NestJS)、`apps/web/admin` (React 管理端) 和 `apps/web/clientWeb` (React 客户端)。
- [x] `@repo/types`, `@repo/ui` 共享包配置。
- [x] 搭建 PostgreSQL 数据库，初始化 Prisma ORM。
- [x] 配置后端的统一响应拦截器、全局异常过滤器。

## 2. 身份验证与后台管理系统（Admin 端）

- [x] 定义用户实体 (User) 与基础角色。
- [x] 密码加密（Bcrypt）。
- [x] 实现完整的 JWT 双 Token 机制。
- [x] 完善注册与登录流程，并在 `admin` 前端对接接口。
- [x] 编写 Guard 实现基于角色的访问控制策略（RBAC）。
- [x] 在管理端 (`apps/web/admin`) 实现用户管理、角色管理以及权限分配等核心功能。

## 2.1 房间系统与文档管理 (核心逻辑层)

- [ ] **数据模型**：定义 `Room` 与 `Document` 实体（房号、标题、创建者、MaxHitz 等）。
- [ ] **后台管理 (Admin)**：
  - 管理员创建/删除房间。
  - 查看全站在线房间列表与人数监控。
  - 房间状态监管（开启/封禁）。
- [ ] **业务逻辑 (API)**：
  - 创建/加入房间权限校验。
  - 房间文档的持久化存储方案（基于 Yjs 的 Binary 或数据库快照）。

## 3. 实时协同编辑系统（Client 客户端）

- [ ] **编辑器端**：在 `apps/web/clientWeb` 中引入 `Tiptap` 构建丰富的富文本编辑器界面。
- [ ] **协同引擎**：引入 `Yjs` 提供 CRDT（无冲突复制数据类型）算法支持，实现毫秒级实时状态共享。
- [ ] **通信层**：前端引入 `y-websocket`，后端提供 WebSocket 支持，建立多人编辑同一个文档的"房间" (Room)。
- [ ] **权限与感知 (Awareness)**：基于 Yjs 广播用户的鼠标光标位置，实现"当前在线用户列表"及"对方光标/姓名高亮展示"功能。

## 4. 性能与持久化优化

- [ ] 将 Yjs 文档树的状态数据（二进制快照）持久化存入 PostgreSQL 数据库中，保证文档内容不丢失。
- [ ] 实现增量同步与全量快照结合的策略，保障长文档的快速加载以及大并发下的性能。
